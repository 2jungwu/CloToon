# Studio Workbench 설계 스펙

## 문서 상태

이 문서는 2026-05-09에 작성된 과거 작업 스펙이다. 현재 기준 문서는 루트의 `PRD.md`, `ROADMAP.md`, `DESIGN.md`이며, 이 파일은 당시 의사결정과 이후 방향 전환을 추적하기 위한 기록으로 보존한다.

2026-05-13 기준으로 폐기되거나 수정된 방향:

- 당시의 `image-only preview` 방향은 폐기되었다. 제작자는 최종 출력에 가까운 자막 박스와 글자 스타일을 미리보기에서 즉시 확인해야 하기 때문이다.
- 자막과 대사를 모두 이미지 생성 참고 정보로만 쓰는 방향은 폐기되었다. 현재 기준은 자막은 HTML/CSS 레이어로 렌더링하고, 대사는 사용자가 입력한 문구에 한해 생성 이미지 안 말풍선/대사 표현으로 반영할 수 있다.
- 컷 세부 설정을 과하게 간소화해 Mock 이미지, 업로드, 복제, 순서 변경을 숨기는 방향은 폐기되었다. 로컬 제작 흐름에서는 Gemini 실패나 quota 문제와 무관하게 mock/업로드 fallback과 컷 관리가 계속 필요하기 때문이다.
- rich text와 자막 박스 편집을 v2로 미루는 방향은 일부 변경되었다. 현재 v1은 기본 자막 레이어 편집기와 컬러 피커, 박스 컨트롤을 포함한다.

## 요약

`Projects`와 `Workspace`를 하나의 제작 화면인 `Studio Workbench`로 통합한다. 사용자는 왼쪽에서 프로젝트를 선택하거나 새 프로젝트를 만들고, 중앙에서 컷 수와 컷별 시나리오/자막/대사/이미지 프롬프트를 편집하며, 오른쪽에서 생성 이미지 미리보기와 내보내기 액션을 확인한다.

이 스펙의 핵심은 카드형 UI를 반복하지 않는 것이다. 화면은 큰 카드 묶음이 아니라 사이드 레일, 프로젝트 패널, 워크벤치 입력 영역, 미리보기 영역의 분할로 구성한다. 각 영역은 그림자나 두꺼운 박스 대신 여백, 얇은 divider, 배경 톤 차이, 활성 상태만으로 구분한다.

## 제품 목표

- 프로젝트 목록 확인, 프로젝트 생성, 프로젝트 삭제를 같은 제작 화면 안에서 처리한다.
- 카드뉴스 프로젝트에서는 전체 시나리오를 입력하고 `한 번에 제작`으로 초안을 만들 수 있게 한다.
- 컷별로 시나리오, 자막, 대사, 이미지 프롬프트를 입력하고 `이미지 생성`으로 현재 컷 이미지를 만든다.
- 자막은 이미지 내부 텍스트가 아니라 HTML/CSS 레이어로 렌더링한다.
- 대사는 사용자가 입력한 문구에 한해 이미지 안 말풍선 또는 자연스러운 대사 표현으로 반영할 수 있다.
- HTML/CSS 기반 preview/export 구조는 유지해 자막 레이어 편집과 출력 동기화를 보장한다.

## 라우팅 방향

- `/projects`는 Studio Workbench의 기본 진입점이다.
- `/workspace/[projectId]`는 기존 북마크 호환용 deep link로 유지한다.
- `/workspace/[projectId]`로 들어오면 같은 Studio Workbench를 열고 해당 프로젝트를 선택한 상태로 표시한다.
- `/assets`는 에셋과 설정 관리 화면으로 유지한다.
- `/settings`는 기존 호환을 위해 `/assets?section=api-key` redirect를 유지한다.

## 레이아웃

### 전체 컨테이너

- 데스크톱 본문 최대 폭은 `1180px`을 기준으로 한다.
- 헤더와 본문 컨테이너 사이 간격은 `80px`을 기준으로 한다.
- 화면 배경은 밝은 near-white를 사용한다.
- 본문은 카드형 외곽 박스로 감싸지 않는다.
- 주요 분할은 vertical divider와 일정한 column gap으로 만든다.

### 1열: 사이드 레일

사이드 레일은 앱의 최상위 이동 구조다.

포함 항목:

- 로고 PNG 자리 표시 영역
- 로고와 메뉴 사이의 수평 라인 1개
- `프로젝트` 메뉴
- `에셋` 메뉴

규칙:

- 로고 영역은 자리만 표시하며 클릭 동작은 없다.
- 수평 라인은 로고 영역 아래에만 둔다.
- 기본 사이드 레일에는 `New` 버튼을 노출하지 않는다.
- 메뉴 항목은 아이콘과 짧은 라벨만 사용한다.
- 현재 선택된 메뉴는 배경 톤과 텍스트 대비로 표시한다.

### 2열: 프로젝트 패널

`프로젝트` 메뉴를 hover하거나 open 상태로 만들면 프로젝트 패널이 열린다. 기존의 `목록` 명칭은 사용하지 않고 패널 타이틀은 `프로젝트`로 통일한다.

포함 항목:

- `프로젝트` 타이틀
- `+ New` 액션
- 프로젝트 리스트
- 프로젝트별 chip metadata
- hover 시 프로젝트 삭제 액션

규칙:

- `+ New`는 프로젝트 패널 안에서만 노출한다.
- 프로젝트 생성 폼은 패널 안에 인라인으로 넣지 않는다.
- 프로젝트 삭제 버튼은 리스트 행 hover 시 또는 워크벤치 head의 명시 액션으로만 노출한다.
- 프로젝트 리스트 행은 카드가 아니라 선택 가능한 row다.
- 선택된 프로젝트 row는 border-heavy 카드가 아니라 배경 톤과 텍스트 색으로 표시한다.

### 3열: 워크벤치 입력 영역

선택된 프로젝트의 제작 입력 영역이다.

상단 head:

- eyebrow `워크벤치`
- 프로젝트 제목
- 콘텐츠 유형 chip
- 캔버스 chip
- 로컬 저장 상태 chip
- 프로젝트 삭제 액션

규칙:

- eyebrow는 모든 화면에서 14px, regular weight로 통일한다.
- 프로젝트 제목은 head 영역 안에 두고, 미리보기 영역이 head 위로 올라오지 않게 한다.
- 프로젝트 삭제 액션은 head의 보조 액션으로 둔다.

카드뉴스 전용 전체 제작 영역:

- 콘텐츠 유형이 `카드뉴스`일 때만 `전체 시나리오` 입력을 표시한다.
- `한 번에 제작` 버튼은 전체 시나리오 아래에 둔다.
- `한 번에 제작`은 사용자가 명시적으로 누를 때만 Gemini API를 호출한다.
- 프로젝트 진입, 프로젝트 선택, 컷 선택만으로 API를 호출하지 않는다.

컷 수 영역:

- 타이틀은 `컷 수`, 16px, medium에 가깝게 표시한다.
- 현재 숫자는 16px, medium에 가깝게 표시한다.
- `준비됨` 상태 텍스트가 필요하면 16px로 표시한다.
- `+`, `-`는 border 없는 icon-only 컨트롤로 둔다.
- `+`를 누르면 컷을 추가한다.
- `-`를 누르면 마지막 컷 또는 선택 컷 삭제 정책에 맞춰 컷을 줄인다.

컷 목록:

- 컷 row는 `#1 첫 장면`처럼 핵심 정보만 표시한다.
- `생성 이미지`, `Mock 이미지`, `이미지 없음` 같은 부제목은 표시하지 않는다.
- 목록은 카드가 아니라 row list다.
- 선택 row만 낮은 배경 톤으로 표시한다.

컷 세부 설정:

- 컷 시나리오
- 자막
- 대사
- 이미지 프롬프트
- `이미지 생성` 버튼

규칙:

- `이미지 생성` 버튼은 이미지 프롬프트 아래에 둔다.
- 템플릿, 이미지 상태, 네거티브 프롬프트는 노출하지 않는다.
- 위로, 아래로, 복제, Mock 이미지, 이미지 업로드 버튼은 노출하지 않는다.
- placeholder는 충분히 흐린 회색으로 조정하되 입력 텍스트와 명확히 구분한다.

### 4열: 이미지 미리보기 영역

이미지 생성 결과를 확인하고 export하는 영역이다.

포함 항목:

- eyebrow `미리보기`
- 미리보기 타이틀
- 회색 placeholder 또는 생성 이미지
- `현재 컷 PNG`
- `전체 ZIP`

규칙:

- 미리보기는 워크벤치 head 아래에 배치한다.
- 현재 기준에서는 미리보기 안에 자막을 HTML/CSS 레이어로 표시한다. 대사는 HTML 오버레이로 표시하지 않는다.
- 이미지가 없을 때는 회색 background와 간결한 empty state만 표시한다.
- 상단 컬러 그라데이션, 무지개 strip, 장식용 shadow는 제거한다.
- export 결과의 HTML/CSS 레이어 유지 여부는 기존 export 호환성을 기준으로 판단한다.

## 프로젝트 생성 모달

`+ New`를 클릭하면 별도 modal을 연다.

필드:

- 프로젝트 이름
- 콘텐츠 유형 dropdown
- 캔버스 dropdown

액션:

- `취소`
- `생성`

시각 규칙:

- modal은 overlay 위에 표시한다.
- modal에는 radius를 적용한다.
- 권장 radius는 `12px`이다.
- modal은 카드형 페이지 구성과 다르게, 일시적 입력 dialog로만 사용한다.
- modal 내부 버튼과 input은 전역 form control 규격을 따른다.

## Chip Component 규격

chip은 한 형태로만 관리한다. 화면마다 다른 badge 스타일을 만들지 않는다.

단일 chip 규격:

- class 예시: `.ui-chip`
- 높이: 24px 안팎
- padding: 좌우 8px
- radius: pill
- font-size: 12px
- font-weight: 500
- 배경: 옅은 blue-tint 또는 neutral-tint 중 하나의 토큰
- 텍스트: primary blue 또는 neutral text 토큰
- border: 기본 없음

사용처:

- 콘텐츠 유형
- 캔버스
- 로컬 저장 상태
- 필요 시 컷 수

금지:

- chip마다 다른 radius를 주는 것
- chip마다 다른 그림자를 주는 것
- chip을 버튼처럼 과하게 강조하는 것
- 같은 metadata를 pill, badge, tag 등 여러 형태로 중복 구현하는 것

## 버튼 규격

- primary는 blue 계열을 유지한다.
- primary 버튼은 주요 생성 액션에만 사용한다.
- `이미지 생성`, `한 번에 제작`, modal의 `생성`이 primary 후보이다.
- secondary 버튼은 neutral 배경과 명확한 텍스트 대비를 사용한다.
- destructive 버튼은 hover 또는 명시 영역에서만 노출한다.
- 저장 버튼류에는 불필요한 아이콘을 넣지 않는다.
- icon-only 버튼은 `+`, `-`, modal close처럼 의미가 즉시 이해되는 경우에만 사용한다.

## 시각 스타일

방향은 밝은 Linear Studio 계열이다.

원칙:

- 배경은 white 또는 near-white 중심이다.
- 분할은 border, divider, 여백으로 만든다.
- 카드형 표면, 강한 그림자, 큰 rounded panel 반복은 피한다.
- primary 색상은 blue 계열을 유지한다.
- active state와 primary CTA에만 blue를 강하게 쓴다.
- 텍스트 대비를 높여 제목, label, 입력값, placeholder의 위계를 명확히 한다.
- 장식용 gradient, 컬러 strip, 불필요한 badge 남발은 제거한다.

## 데이터와 API

이 스펙은 UI composition 변경이다.

유지:

- 기존 `Project` 타입
- 기존 `Cut` 타입
- 기존 SQLite schema
- 기존 프로젝트 API
- 기존 컷 API
- 기존 `local-studio-assets`
- 기존 `local-studio-settings`
- 기존 PNG/ZIP export 구조

허용되는 보조 변경:

- Workbench 화면 조합을 위한 client-side state 정리
- `/projects`와 `/workspace/[projectId]`가 같은 workbench component를 공유하도록 하는 컴포넌트 분리
- `한 번에 제작`과 `이미지 생성` 호출을 명확히 나누는 handler 정리

금지:

- API key를 DB, 파일, git, 로그에 저장
- 페이지 진입만으로 Gemini 호출
- 사용자의 명시 액션 없이 외부 provider 호출
- 데이터베이스 초기화 또는 기존 프로젝트 삭제

## Gemini 생성 UX

`이미지 생성`:

- 현재 선택 컷만 대상으로 한다.
- 컷 시나리오, 자막, 대사, 이미지 프롬프트를 함께 사용한다.
- Assets의 캐릭터/배경 설정을 함께 사용한다.
- 자막은 이미지에 글자로 넣지 않고 HTML/CSS 레이어로 렌더링한다.
- 대사는 사용자가 입력한 문구에 한해 이미지 안 말풍선 또는 대사 표현으로 반영할 수 있다.
- 생성 중, 성공, 실패 상태를 버튼 근처에 표시한다.

`한 번에 제작`:

- 카드뉴스 프로젝트에서만 사용한다.
- 전체 시나리오와 컷 수를 기준으로 컷 초안을 생성한다.
- API 호출 폭증을 막기 위해 순차 처리하거나 명확한 진행 상태를 제공한다.
- 인스타툰 프로젝트에서는 표시하지 않는다.

## 반응형

데스크톱:

- 사이드 레일
- 프로젝트 패널
- 워크벤치 입력 영역
- 이미지 미리보기 영역

태블릿:

- 사이드 레일은 유지한다.
- 프로젝트 패널은 접힘 또는 overlay drawer로 전환할 수 있다.
- 입력 영역과 미리보기 영역은 2열 또는 세로 흐름으로 전환한다.

모바일:

- 사이드 레일은 상단 또는 하단 navigation으로 전환할 수 있다.
- 프로젝트 패널은 drawer로 표시한다.
- 입력 영역과 미리보기는 세로로 쌓는다.
- 텍스트와 버튼은 겹치지 않아야 한다.

## v2 메모

Genspark식 고급 직접 편집은 v2에서 다룬다. 현재 v1은 자막 텍스트 툴바, 컬러 피커, 자막 박스 컨트롤까지 포함한다. 다만 미리보기 영역에서 텍스트나 이미지 요소를 직접 선택하고 floating toolbar로 이동/리사이즈/undo/redo까지 제공하려면 별도의 layer/document model, selection state, undo/redo 정책, export 동기화 정책이 필요하다.

## 사용 기준

- 기본 사이드 레일에는 로고 자리, 구분선, `프로젝트`, `에셋`만 보인다.
- `New`는 프로젝트 패널 안에서만 보인다.
- `New` 클릭 시 radius가 있는 생성 modal이 열린다.
- 프로젝트 리스트는 row list이며 카드처럼 보이지 않는다.
- chip은 모든 화면에서 하나의 형태로 보인다.
- 워크벤치 head 아래에 입력 영역과 미리보기 영역이 배치된다.
- 미리보기는 생성 이미지와 HTML/CSS 자막 레이어를 함께 보여준다. 대사 HTML 오버레이는 표시하지 않는다.
- 카드뉴스 프로젝트에서만 전체 시나리오와 `한 번에 제작`이 보인다.
- 인스타툰 프로젝트에서는 전체 시나리오 batch 영역이 보이지 않는다.
- 컷 수 조절의 `+`, `-`는 border 없는 icon-only 컨트롤이다.
- 컷 목록에는 부제목이 보이지 않는다.
- 기존 프로젝트, 컷, Assets, Settings, export 데이터는 유지된다.
