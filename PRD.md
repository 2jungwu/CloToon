# 로컬호스트 기반 인스타툰/카드뉴스 제작 스튜디오 MVP PRD

## 1. 핵심 정보

**목적**: 개인 창작자가 로컬 PC에서 시나리오를 컷으로 나누고, 자막/대사/프롬프트/이미지를 편집해 SNS용 인스타툰과 카드뉴스를 PNG로 제작하게 한다.  
**사용자**: 캐릭터 일관성과 텍스트 품질을 직접 통제하며 반복적으로 인스타툰/카드뉴스를 만드는 1인 창작자.

## 2. 검증 기반 제품 결정

- [FACT] Fastify는 `host: '127.0.0.1'` 지정 실행을 지원하므로 localhost 전용 앱 구조는 기술적으로 가능하다.
- [FACT] Gemini API는 API 키 기반 호출과 이미지 생성 모델 사용을 공식 문서로 제공한다.
- [FACT] Gemini 구조화 출력은 JSON Schema 기반 응답을 지원하지만 JSON Schema 전체가 아니라 일부만 지원한다.
- [FACT] Playwright는 페이지 스크린샷과 screenshot assertion을 지원하므로 HTML/CSS 컷을 PNG로 검증하는 흐름에 적합하다.
- [INFERENCE] 최종 원본을 HTML/CSS로 두고 AI 이미지를 텍스트 없는 장면 이미지로 제한하면 한글 자막/대사 품질을 안정적으로 통제할 수 있다.
- [UNCERTAIN] 실제 Gemini 이미지 생성 가능 모델, 무료 한도, 지역 제한, 과금 상태는 사용자 API 키와 Google 정책에 따라 달라지므로 앱은 무료 생성을 보장하지 않는다.

## 3. 사용자 여정

```text
1. 앱 실행
   ↓ 로컬 서버 시작

2. 프로젝트 목록 페이지
   ↓ 새 프로젝트 생성 또는 기존 프로젝트 열기

3. 프로젝트 워크벤치
   ↓ 콘텐츠 유형 선택: 인스타툰 / 카드뉴스
   ↓ 입력 방식 선택
      전체 시나리오 입력 → 컷 초안 생성
      컷 수 지정 입력 → 컷별 시나리오 작성
   ↓ 컷 편집

4. 자산 설정 페이지
   ↓ 캐릭터 / 배경 / 폰트 등록 후 워크벤치로 복귀

5. 이미지 생성 패널
   ↓ mock provider / Gemini provider / 직접 업로드 중 선택
   ↓ 컷 이미지 적용

6. 미리보기 및 다운로드
   ↓ 현재 컷 PNG 또는 전체 ZIP 다운로드

7. 완료
   → 프로젝트 계속 편집
   → 새 프로젝트 생성
```

## 9. 수용 기준

### 기본 제작 흐름

- 사용자가 로컬 앱을 실행하고 `http://localhost:3000`에서 프로젝트 목록 페이지를 볼 수 있다.
- 사용자가 새 프로젝트를 만들고 인스타툰 또는 카드뉴스 유형을 선택할 수 있다.
- 사용자가 전체 시나리오 입력 또는 컷별 입력 중 하나를 선택할 수 있다.
- 사용자가 컷의 자막, 대사, 이미지 프롬프트, 네거티브 프롬프트를 수정할 수 있다.
- 사용자가 인스타툰 템플릿과 카드뉴스 템플릿의 서로 다른 HTML 미리보기를 확인할 수 있다.

### 캐릭터와 자산 흐름

- 사용자가 `clo` 캐릭터를 만들 수 있다.
- 사용자가 `clo.md` 설명 파일과 표정별 이미지를 업로드할 수 있다.
- 컷에서 `clo`와 특정 표정을 선택하면 이미지 프롬프트 초안에 반영된다.
- 폰트 프로필 변경 시 HTML 미리보기의 자막/대사 스타일이 갱신된다.

### 이미지 생성 흐름

- 사용자가 API 키 없이 mock provider로 이미지 적용 흐름을 끝까지 테스트할 수 있다.
- 사용자가 Gemini API 키 사용 방식을 설정 페이지에서 확인할 수 있다.
- Gemini 호출 전 전송될 데이터 범위를 이미지 생성 패널에서 확인할 수 있다.
- Gemini 호출 실패 시 오류가 표시되고 직접 업로드 fallback을 선택할 수 있다.

### 다운로드 흐름

- 현재 컷을 기본 `1080x1080` PNG로 다운로드할 수 있다.
- 전체 컷을 ZIP으로 다운로드할 수 있다.
- export 전 폰트 로딩 완료를 기다린다.
- 텍스트가 컷 영역을 넘치면 다운로드 전에 경고를 표시한다.
- 다운로드된 PNG에는 생성/업로드 이미지, 자막, 대사, 폰트 스타일이 모두 반영된다.

## 10. 테스트 계획

- Unit: 시나리오 입력을 컷 JSON으로 변환하는 로직을 검증한다.
- Unit: 캐릭터 Markdown과 표정 메타데이터가 프롬프트 초안에 반영되는지 검증한다.
- Unit: FontProfile이 HTML 스타일로 변환되는지 검증한다.
- Integration: mock provider로 프로젝트 생성, 컷 편집, 이미지 적용, PNG export 흐름을 검증한다.
- Integration: SQLite 저장 후 프로젝트와 컷을 다시 불러오는지 검증한다.
- E2E: `clo` 캐릭터를 등록하고 인스타툰/카드뉴스 각각 PNG를 다운로드한다.
- E2E: 다운로드 결과가 `1080x1080`이고 텍스트가 HTML 레이어로 표시되는지 확인한다.
- Optional: 실제 Gemini API 호출은 별도 수동 테스트로 분리하고 기본 테스트에서는 실행하지 않는다.

## 11. MVP 정합성 체크

- 기능 명세의 모든 기능 ID는 페이지별 상세 기능에 존재한다.
- 메뉴 구조의 모든 항목은 페이지별 상세 기능에 존재한다.
- 페이지별 상세 기능의 모든 구현 기능 ID는 기능 명세에 정의되어 있다.
- `F001`부터 `F014`까지 구현 페이지가 하나 이상 연결되어 있다.
- 실제 API 비용 없이 mock provider로 기본 제작 흐름을 검증할 수 있다.
